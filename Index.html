The MACRO Project

Mars Cargo Mobility System: An autonomous rover designed to traverse simulated Martian terrain, detect magnetic beacons, and deliver delicate cargo.

Project Abstract

The Challenge: Design an autonomous robot to pathfind along a black line, detect magnetic beacons, and traverse obstacles to deliver three distinct cargo types without tipping.


The Solution: We engineered a compact Rear-Wheel Drive (RWD) rover featuring a custom "hugger" cargo mechanism. The system utilizes sensor fusion (Ultrasonic + IMU) and a Raspberry Pi control unit to navigate and adapt to terrain changes in real-time.

Mechanical & System Design

Drivetrain Evolution

We iterated through three distinct drivetrain designs. We rejected Treads due to axle bending and Front-Wheel Drive due to poor traction on inclines. We finalized on a Rear-Wheel Drive (RWD) system which centralized mass over the drive motors for maximum grip.

The "Hugger" Mechanism

To secure cargo without complex hydraulics, we designed a worm-gear driven "hugger." It clamps the cargo during transport and retracts to release it. A secondary "pusher" wall ensures the payload is ejected cleanly at the drop zone.

Sensor Integration

The robot features a multi-sensor array:
• IMU (Gyro): For magnetic beacon counting and dynamic power adjustment on ramps.
• Ultrasonic: For obstacle avoidance (< 10cm).
• Line Finders: Dual IR sensors for PID-based path correction.

Autonomous Navigation Logic

The software utilizes a state-machine architecture. Below is the core search logic used when the robot loses the line. Instead of stopping, it initiates a "Look Left / Look Right" pattern to recover the path.

def main():
    motorA = Motor('C')
    motorB = Motor('D')
    lineFinderA = LineFinder(24) # Right Sensor
    lineFinderB = LineFinder(26) # Left Sensor

    while True:
        # Standard Line Following
        if curValueA == 1 or curValueB == 1:
            goForward(motorA, motorB, 0.3, 0.25)
            continue
          
        # Lost Line Recovery Logic
        elif curValueA == 0 and curValueB == 0:
            print("No line detected. Searching...")
            
            # 1. Check Left
            turnLeft(motorA, motorB, .2, 1)
            if lineFinderA.value == 1 or lineFinderB.value == 1: 
                goForward(motorA,motorB, 0.3, .25)
                continue
            
            # 2. Check Right (wider sweep)
            else: 
                turnRight(motorA, motorB, .2, 2)
                if lineFinderB.value == 1:
                    goForward(motorA, motorB, 0.3, .25)
                    continue

Written in Python using the BuildHat library for Raspberry Pi.

Critical Analysis & Reflection

The "Demo Day" Anomaly

While the rover performed perfectly in pre-demo testing (completing 5/6 tasks), the final demonstration faced challenges. We identified two root causes:

  • Ambient Light Interference: The high reflectivity of the gymnasium floor blinded the IR sensors, which were calibrated for the dimmer dorm lounge.
  • Motor Hysteresis: The motors struggled to reverse instantly during high-frequency switching, causing the robot to jitter rather than turn.

Lesson Learned: Future designs will include a robust "Calibration Mode" to adjust sensor thresholds on-site and a software buffer to prevent rapid motor direction switching.
